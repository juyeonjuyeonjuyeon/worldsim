# CLAUDE.md — WS Forest Weather 작업 규칙

> 이 파일은 Claude Code가 매 세션 자동으로 읽고 따르는 영구 규칙이다.
> **자동(무인) 실행 시 특히 엄격하게 적용한다.**

---

## 0. 언어
- 모든 응답·로그·문서·커밋 메시지는 **한국어**. 영어 금지.

## 1. 절대 변경 금지 — 검증 완료, 동결
사람 눈으로 여러 번 검증해 통과한 것들. **숫자가 맞아 보여도 건드리지 않는다.**
- 천문 위치 계산: 태양·달·행성의 적위/고도/방위, 낮 길이, 대기 굴절(Bennett),
  세차(IAU1976), GMST, `v_world_dir_y` 지평선 클립.
- 태양·달 디스크 각지름(태양 0.53°, 달 0.52°)과 글레어.
- 노출/톤매핑 파이프라인 로직 자체.
- 달 메쉬 셰이더(위상 마스크).

## 2. 물리 모델 우선 — 핵심 철학
- 모든 현상은 **근본 물리에서 창발**해야 한다(원인→결과). 색·값을 손으로 칠하거나
  스크립트로 발생시키지 않는다.
- 목표 정확도 = "단순화했지만 **올바른 정성적 인과를 보존**하는 물리". 슈퍼컴 정밀도 불필요.
- `[수동]` 분류 항목은 물리/절차 모델로 마이그레이션 대상.
  최우선 = 하늘 색(박명·태양 색온도·달 적화) → **대기 산란 모델**.

## 3. 검수 — 자가 스크린샷 + 사람 최종 확인
렌더·셰이더·색에 영향을 주는 변경 후에는 **반드시**:
1. 헤드리스로 해당 조건(시각·날씨)을 렌더해 **스크린샷 PNG로 저장**.
2. 그 PNG를 **직접 열어(Read 도구) 명시된 통과/실패 기준과 대조**.
3. 판정과 근거(무슨 색으로 보이는가)를 로그에 기록.

**비전 자가 검수의 한계 — 반드시 명심:**
- 비전은 **"명백한 실패"**(갈색 박명, 칠흑/과밝은 밤, 깨진 렌더, 잘린 천체)를 거르는
  **1차 필터일 뿐**이다.
- 통과해도 **"완료"로 단정하지 말고 `사람 최종 확인 대기`로 표시**한다.
  미묘한 색감·깊이·분위기는 **사람 눈이 최종 기준**.
- 컴파일 통과·테스트 green 같은 **수치만으로 시각 결과를 "완료" 단정 금지.**
  (밤 암막 버그가 이렇게 여러 번 "고쳤다"고 한 뒤 재발했다.)

## 4. 안전 — 커밋 격리 + 되돌리기
- 작업 시작 전 **체크포인트 커밋**.
- **한 목표 = 한 독립 커밋.** 다른 목표 작업과 섞지 않는다(되돌리기 쉽게).
- 매 변경 후 `godot --headless`로 **파스/컴파일 통과 확인**.
- 컴파일 깨짐 또는 검수 **명백한 실패** → 그 목표의 변경만 **되돌리고(reset)**,
  로그에 남기고, **다음 목표로 넘어간다. 막혀서 멈추지 말 것.**
- 불확실하면 절반 구현 대신 **TODO 스텁 + `사람 확인 필요`**.

## 5. 진단 우선
- 버그는 추측으로 고치지 말고 **원인을 먼저 진단·기록한 뒤** 수정한다.
- 검증 FAIL이 나면 **기준(기대값·허용오차)을 느슨하게 바꾸지 말고 시뮬레이션 쪽을 의심**한다.

## 6. 범위 단계 (LOD)
- 웹·앱·PC 동등 배포 목표, **모바일이 성능 기준선**.
- 신규 효과·셰이더는 품질 단계(고=PC / 중=앱 / 저=웹)로 만들 수 있게 설계.

## 7. 기록 의무
- 자동/무인 실행 중에는 **진행 로그를 계속 갱신**한다. 각 목표마다:
  (a) 시도한 것 (b) 진단 (c) 변경 내용 (d) 스크린샷 판정 + 근거
  (e) 통과/실패 (f) 남은 일.
- **중간 과정 로그 + 완료 요약을 둘 다** 작성한다. (구체 파일명은 실행 지시서에서 지정.)
