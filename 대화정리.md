# WorldSim 프로젝트 대화 정리
날짜: 2026-06-16

---

## 목표

상상한 세계 공간을 물리 엔진 기반으로 시뮬레이션하고, 시간의 흐름에 따른 변화를 관찰하는 앰비언트 영상을 만든다.
AI가 생성한 그럴싸한 영상이 아닌, 실제 물리 법칙에 기반한 시뮬레이션.

---

## 만들고 싶은 세계관

1. **숲**
   - 계절과 날씨별 시간 변화
   - 상상의 공간 또는 실제 나라별 기후/지형 특성 모방 (열대우림, 침엽수림, 단풍림 등)

2. **우주/행성**
   - 화성에서 노을 보기 (실제 화성 대기 물리 기반)
   - 특정 행성 또는 가상의 행성 (중력, 대기 조성, 지표 물질 설정)
   - 참고: 화성 노을은 낮엔 붉고 해질 때 파랗게 변함 (지구와 반대)

3. **조선시대 한옥 24시간 관찰**
   - 처마 밑으로 스며드는 빛, 마당의 그림자 변화 등

## 도구 선택

### 검토한 도구들
- **Genesis (Genesis-Embodied-AI)**: 로봇 시뮬레이션 특화. 대기 산란, 구름, 자연환경 미지원 → 부적합
- **Unreal Engine 5**: 강력하지만 오브젝트 직접 배치 필요, GUI 중심 → 진입장벽 높음
- **Blender + Python**: 물리 기반 렌더러 + 대기 모델 + Python 완전 제어 → 선택

### Blender를 선택한 이유
- Python(bpy)으로 모든 것을 코드로 제어 가능
- 물리 기반 대기 산란 모델 내장 (Blender 5.1 기준: MULTIPLE_SCATTERING)
- Cycles 레이트레이싱 렌더러 (빛 산란, 그림자, 대기 효과 자동 계산)
- 완전 무료 및 오픈소스
- 헤드리스 실행 가능 (GUI 없이 스크립트만으로 렌더링)

## 구현 방식

### 전체 흐름
```
파이썬 스크립트 작성
    ↓
Blender 헤드리스 실행
    ↓
장면 자동 생성 (지형, 나무, 에셋 절차적 배치)
    ↓
물리 기반 시뮬레이션 (대기, 조명, 날씨)
    ↓
렌더링 → 영상 출력
```

### Blender 실행 명령어
```powershell
& "C:\Program Files\Blender Foundation\Blender 5.1\blender.exe" --background --python "스크립트경로.py"
```

### 구현 가능한 요소
| 요소 | 방법 |
|------|------|
| 대기 산란 | MULTIPLE_SCATTERING (Blender 5.1 물리 모델) |
| 구름 | 볼류메트릭 + 노이즈 텍스처 |
| 안개 | 볼류메트릭 스캐터링 |
| 숲 생성 | 절차적 생성 (규칙 정의 → 자동 배치) |
| 24시간 사이클 | 태양 각도 애니메이션 (프레임 = 시간) |
| 빛 산란 | Cycles 레이트레이싱 (자동) |
| 계절 변화 | 나뭇잎 색상/밀도 애니메이션 |
| 날씨 | 비, 눈, 안개, 구름 파티클 시스템 |

### 시간 표현 방식
- Blender 프레임 = 시간 단위
- 예: 프레임 1 = 0시, 프레임 240 = 24시간
- 태양 각도(sun_elevation)를 시간 함수로 애니메이션

## 기술 노트: Shape Key 실패 원인 분석

Shape Key 방식이 렌더에서 비가 안 보인 이유:
- Blender `bpy.ops.render.render()` 내부에서 depsgraph가 재평가됨
- Shape Key 애니메이션 값이 `frame_change_post` 이후에도 depsgraph 캐시에서 계산됨
- Python으로 Shape Key 값을 직접 설정해도 렌더 시점에 override될 수 있음
- `frame_change_post` 핸들러 + `update_tag()` 조합이 확실한 해결책

## 기술 노트: 비 렌더링 방법 비교

| 방법 | 결과 | 문제 |
|------|------|------|
| Vertex Instancing | 까만 점만 표시 | Cycles에서 인스턴스 렌더 실패 |
| Particle System | 비 안 보임 | 백그라운드에서 파티클 DEAD (중력 > 수명) |
| Geometry Nodes | 비 안 보임 | 백그라운드 렌더에서 작동 안 함 |
| Shape Key 애니메이션 | 비 안 보임 | depsgraph 캐시가 Python 변경 무시 |
| **Curve + 프레임 핸들러** | **비 선명하게 보임** | **없음 → 채택** |

### 채택 방식 상세
- `bpy.app.handlers.frame_change_post.append(fn)`: 매 프레임 변경 시 Genesis 좌표를 Curve에 직접 기록
- `rain_obj.data.update_tag()`: depsgraph에 변경 알림 (렌더 반영 필수)
- Curve `bevel_depth=0.012` (12mm) + Emission 0.4: 배경 무관하게 가시성 보장
