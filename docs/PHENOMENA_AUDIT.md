# 천문·기상 현상 감사 (Phenomena Audit)

코드베이스 기준: 2026-06-29  
감사 범위: `Astronomy.gd`, `Sky.gd`, `Environment.gd`, `Weather.gd`, `Main.gd`, `Sound.gd`

---

## 물리 분류 기준

| 기호 | 의미 | 판단 기준 |
|------|------|---------|
| **[물리]** | 근본 물리·천체역학 | 입력(태양 위치·수증기 등)을 빼면 결과도 사라진다. 공식 자체가 자연법칙. |
| **[절차]** | 물리 기반 근사·확률 모델 | 입력에 반응하지만, 실측 테이블·확률표·보간 공식으로 계산. 완전한 물리는 아님. |
| **[수동]** | 고도·시각별 색·값 직접 지정 | 입력 변화에 반응은 하지만, 핵심 값(색·계수·단계)이 코드에 박혀 있다. 회귀 위험 높음. |

| 암순응 (scotopic) 모드 | Sky.gd | ✅ | **[수동]** ⚠️ | `scotopic_boost=4.0`, `scotopic_r=0.80`, `scotopic_g=0.90`, `scotopic_b=1.15` 전부 수동. eye_view 플래그 on/off만, 실제 조도(lux)와 연동되지 않음 — 조도가 변해도 계단식 전환. → 마이그레이션: 조도 기반 Purkinje 이동 공식 (CIE 스펙트럼 감도 곡선) |
| 달 위상 이미지 | Sky.gd | ✅ | **[물리]** | 셰이더에서 sun_dir × 달 법선 내적으로 밝은면/어두운면 결정. 태양 방향 제거 시 달 전체 어두워짐. |

| **토성 고리** | 행성 렌더 확장, 고리 경사각, 조도 | 상 | Phase 1 | 현재 Venus·Jupiter만 구체; Saturn은 카탈로그 점 |

| **백야/극야** | 위도 > 66.5°, 날짜 → 태양이 지지 않거나 뜨지 않음 | 중 | Phase 1 | Sky.gd의 태양 고도 상시 > 0 or < 0 처리 필요 |
| **ISS·위성 통과** | TLE 궤도 요소, 실시간 또는 에폐머리스 | 상 | Phase 2+ | 외부 API 또는 SGP4 적분; 이동 점광원 |
| **소행성** | Meeus 궤도 요소 추가 | 하 | Phase 1 | 혜성 코드 패턴 재사용; 수작업 데이터 |

| **신기루** (Inferior Mirage) | 지표 온도 > 공기 온도 (강한 가열), 평탄 지형 | 상 | Phase 3 | 지평선 "물웅덩이" 역상 굴절; 지형 데이터 필요 |
| **상위 신기루** (Superior Mirage) | 해상 온도 역전층 | 상 | Phase 3 | 수평선 너머 물체 보임 |
| **파타 모르가나** (Fata Morgana) | 다층 온도 역전 | 상 | Phase 3 | 다중 역상·압축 이미지 |
| **박명광선** (Crepuscular Rays) | 구름 틈새, 볼류메트릭 산란 | 상 | Phase 3 | "신의 광선"; 볼류메트릭 구름과 연동 |
| **반황혼선** (Anti-crepuscular Rays) | 수평선 반대편 수렴 | 상 | Phase 3 | 하늘 가로질러 반대쪽 수렴점 |
| **열기 아른거림** (Heat Haze) | 지면 온도 > 30°C, 직사광선 | 중 | Phase 3 | 굴절 왜곡 포스트프로세스; 화면 공간 |
| **성층권 에어로졸** | 화산·산불·인위 AOD 입력 | 상 | Phase 3 | 하늘 탁도 증가, 일몰 붉어짐 |

### 2-3. 강수·수문 현상

## 3. [수동] 항목 마이그레이션 우선순위

[수동]으로 분류된 7개 항목을 회귀 위험·영향 범위·구현 용이성 기준으로 정렬.

### 우선순위 1 — 하늘 색 테이블 (박명 4단계 + 태양 색온도)

**항목**: `박명 4단계` · `태양 색온도 (고도별)`  
**현재 코드**: Sky.gd 896–921번 줄 (`day_top`, `sunset_top`, `civil_top`, `naut_top`, `day_horizon`, `sunset_horizon`, `civil_hor`, `naut_hor`); 821–831번 줄 (`warm`, `white/orange/moonlt`)  
**회귀 이력**: 밤 암막 버그, 박명 갈색 버그 모두 이 수동 색 값 + 노출 보정 충돌에서 발생.  
**마이그레이션 목표**: Hosek-Wilkie 분석 모델 → `sky_scatter.gdshader` 커스텀 스카이  
**의존 설계 문서**: [SKY_DESIGN.md](SKY_DESIGN.md) — Step 1(LOD 0 ProceduralSky에 Hosek-Wilkie 출력 주입)부터 단계적 교체  
**Phase**: 1 (현 구조, 리스크 최소 경로 있음)

---

### 우선순위 2 — 달 대기 적화

**항목**: `달 대기 적화 (고도별)`  
**현재 코드**: Sky.gd 803–813번 줄 (`moon_warm`, `lerp(0.62, 0.28, moon_warm*0.65)` 등)  
**문제**: 18° 기울기 임의; 실제 대기 광로 길이와 무관. 박명 시 달색이 하늘색과 불일치 가능.  
**마이그레이션 목표**: 달 고도에서 대기 광로 길이(secz 근사) + 레일리 산란 파장별 투과율  
**Phase**: 1 (Hosek-Wilkie 도입과 동시에 처리 가능)

---

### 우선순위 3 — 암순응 (scotopic) 모드

**항목**: `암순응 (scotopic) 모드`  
**현재 코드**: Sky.gd 869–878번 줄 (`scotopic_boost=4.0`, `scotopic_r=0.80`, `scotopic_g=0.90`, `scotopic_b=1.15`)  
**문제**: eye_view on/off 계단식 전환; 조도(lux)가 아무리 변해도 야간 한복판이든 달밤이든 똑같이 4× 부스트. 보름달 밝은 밤과 신월 어두운 밤의 시각적 차이를 놓침.  
**마이그레이션 목표**: `total_lux`를 입력으로 받아 Purkinje 이동 정도를 연속 계산. 예: `scotopic_weight = clamp(1 - total_lux / 0.3, 0, 1)` + CIE 스코토픽 상대 감도 곡선(V'(λ)) 적용  
**Phase**: 1 (조도 계산이 이미 있으므로 연동만 하면 됨)

---

### 우선순위 4 — 구름 형태

**항목**: `구름 단일 레이어`  
**현재 코드**: Sky.gd `_update_cloud_visual` (UV 스크롤 노이즈 고정)  
**문제**: CIRRUS(얇은 실 형태)·CUMULUS(뭉게구름)·OVERCAST(층운)가 동일 형태. 형태가 날씨 조건에 반응하지 않음.  
**마이그레이션 목표**: 볼류메트릭 구름 (CLOUD_DESIGN.md) — Step 1은 날씨 유형별 텍스처 분화(LOD 0 개선)  
**Phase**: 3 (볼류메트릭 이전에는 날씨별 텍스처 스왑으로 부분 개선 가능)

---

### 우선순위 5 — 사운드: 비·눈·바람

**항목**: `사운드: 비/눈/바람`  
**현재 코드**: Sound.gd WAV 루프 5개 (rain_light/medium/heavy + wind_loop + snow_loop)  
**문제**: 재질 무관 (숲·지면·웅덩이 동일 소리), 공간 잔향 없음, 연속 강도 전환 시 클릭  
**마이그레이션 목표**: AUDIO_DESIGN.md Phase 4-1~4-3 (잔향 버스 → 연속 크로스페이드 → 재질별 채널)  
**Phase**: 4 (귀 검수 필수)

---

### 우선순위 6 — 사운드: 천둥

**항목**: `사운드: 천둥`  
**현재 코드**: Sound.gd WAV 원샷 3개 (close/mid/far)  
**문제**: 거리가 연속값이어도 3단계 이산. 기온·습도에 따른 음색 변화 없음.  
**마이그레이션 목표**: AUDIO_DESIGN.md Phase 4-5 (거리 연속 크로스페이드 → 합성 전환)  
**Phase**: 4

---
